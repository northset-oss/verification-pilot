CLAIMS BOUNDARY

What a receipt does and does not say

A receipt reports the exact declared commands, their recorded exit status, and the recorded execution environment from an immutable run bundle.

It does not prove code quality, security, full CI coverage, production readiness, or maintainer approval. An attestation confirms bundle provenance; it does not broaden the receipt's claim.

Live upstream outcomes are detached because a later maintainer decision is a different fact from the recorded run. Read the full Claims Boundary policy.

SELF-FUNDED FIELD-TESTING. A proof-of-pass receipt records that the declared commands returned exit 0 on the named code in the named environment.
